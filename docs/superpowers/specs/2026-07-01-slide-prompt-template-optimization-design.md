# 兩份 Gamma Slide Prompt Template 優化設計

> 對象：`templates/1_to_n_slide_prompt_template.md`（既有產品功能更新）、`templates/0_to_1_slide_prompt_template.md`（全新產品提案）
> 目的：使用者貼上 spec.md + plan.md 後，模板產出一份可直接貼進 Gamma AI 簡報生成軟體的 Prompt（含大綱、演講稿、雙重稽核官審查報告）。

---

## 1. 問題陳述

現有兩份 template 存在五個具體問題：

1. **視覺風格與 `pm_proposal_presentation.pptx`（標準答案示範簡報）落差**：配色選錯色階、缺頁尾/Overline、流程圖用 SVG/CSS 語法（Gamma 讀不懂）。
2. **缺標題頁**：兩份 template 都直接從內容頁開始，沒有封面頁規格。
3. **缺嚴謹的 agentic 執行架構**：現有「🤖 智能體設計模式執行指南」提到的 pattern 名稱不精確，稽核關卡缺乏理論依據與停止條件。
4. **主產品/北極星指標推論常失敗**：現有機制是「5 隻虛擬 Agent 共識會議 + 事後警告標籤」，本質是自欺式表演，沒有真正向使用者確認的機制。
5. **兩份 template 內容大量重複，維護成本高**。

## 2. 調研依據摘要

本設計基於 7 份調研報告的結論（6 份並行 + 1 份追加）：

| # | 調研主題 | 核心結論 |
|---|---|---|
| 1 | pptx 視覺語法拆解 | 標題頁/內容頁/流程圖頁有精確版式公式；強調色實測為 `#007583`（非 template 原用的 `#48B7BD`）；流程圖是 PowerPoint 原生形狀（圓形節點、色塊卡、freeform 折線），非 SVG |
| 2 | agentic-design-patterns repo | 雙重稽核官對應 **Reflection Pattern**（生產者/評審者角色分離，避免同套邏輯自我審查的認知偏差）+ **Multi-Agent Collaboration 的 Critic-Reviewer/Debate & Consensus**；Human-in-the-Loop 應用「具體情境枚舉」取代模糊信心分數；Planning Pattern 支持「固定工作流優於動態規劃」與「先骨架後全文」的強制兩段輸出 |
| 3 | 澄清提問技術主流解法 | 現有「5-Agent 虛擬共識會議」應廢除；改採**信心分級**（High/Medium/Low），低信心才停止並提問（一次問完、批量），中高信心才推論並保留警告標籤作次要防線 |
| 4 | NNgroup 低摩擦互動原則 | **先給 AI 推論當預設答案讓使用者確認/修正**優於開放式提問（Anchoring Principle 實證 +23% 完成率）；一次最多問 1-2 個關鍵問題、用選項代替開放問答、不重複盤問、問前先講為什麼問 |
| 5 | Gamma AI 能力與限制 | Gamma **不會**解析文字裡的 hex 色碼（需用 Theme/Brand Kit UI 設定）、**不會**渲染 inline SVG、生成中**無多輪澄清機制**——所有澄清必須在貼進 Gamma 之前完成 |
| 6 | 真實 spec/plan 範例分析（cryptosense 案例） | P1/P2/P3 分期用詞會誤導模型誤判「有主產品」；北極星指標常「有定義、無基準數字」；「簡報受眾/用途」「目前專案階段」是文件天生不會回答的資訊缺口 |
| 7 | （追加）Gamma markdown 格式精確度 | Gamma 是「AI 生成引擎」而非 Slidev/Marp 那種「渲染引擎」；連 API 最保守的 `preserve` 模式官方都寫明「仍可能自行加結構」；**不應**在 markdown 裡用表格/巢狀清單模擬版面，該用語意標籤 + Outline-first 兩步審閱流程 |

## 3. 設計決策

### 3.A 澄清機制：信心分級 Human-in-the-Loop（取代 5-Agent 虛擬共識）

廢除虛擬共識會議。改為：

```
1. 分析 spec.md/plan.md，針對 5 個關鍵欄位各自評信心（High/Medium/Low）：
   主產品定位(0-to-1 vs 1-to-N，注意 P1/P2/P3 等分期用詞不代表既有主產品)
   ｜北極星指標定義｜北極星指標基準值/目標值｜簡報受眾與用途｜目前專案階段

2. 任一欄位 = Low → 停止產出，一次性列出所有 Low 欄位的澄清問題（≤5 題），
   每題附「AI 推論候選值，請確認或修正」（Anchoring：預設答案+確認，而非
   純開放式問答），問題前先講一句「為什麼問這個」。等待使用者回覆後才繼續。
   使用者可回「不用問我，直接用你的推論」跳過此步驟（一次性 opt-out）。

3. Medium/High → 直接推論並寫出推論依據（一句話說明根據文件哪處線索），
   保留 ⚠️ 警告標籤作為次要防線。不執行虛擬多角色表演。
```

- 不加「快速通道」（跳過信心自評）：每次都執行完整評估，避免誤判導致漏問（使用者已確認的決策）。
- 依 Human-in-the-Loop Pattern 的「升級策略」精神，Low 觸發條件用**具體情境枚舉**（缺哪個欄位、矛盾在哪），不是模糊的「不確定時」。

### 3.B 雙重稽核官：升級為具名 Pattern

- 草擬者／稽核官 A（內容真實性與邏輯）／稽核官 B（視覺與敘事）三個角色明確互斥分工，各自的評估標準寫死在 prompt 裡，不是「請再檢查一次」。
- 兩位稽核官意見衝突時，走 **Debate & Consensus**：雙方各自陳述理由，草擬者裁決並記錄依據。
- 明確**停止信號**（雙方皆 PASS），**最多 2 輪**，超過標記「需人工判斷」並列出爭議點——避免無限迴圈與 token 浪費。
- 大綱先用「### 骨架 → ### 完整內容」兩段式強制輸出（Planning Pattern 的 CrewAI 式結構化輸出技巧），避免邊寫邊想。
- 在「🤖 智能體設計模式執行指南」章節明確標註對應的具名 pattern（Prompt Chaining / Planning / Reflection / Multi-Agent Collaboration / Human-in-the-Loop），取代目前含糊的描述。

### 3.C 新增標題頁（Slide 0）

兩份 template 的輸出格式最前面新增一頁封面，依 pptx 精確版式：

- 全頁暖米白底色。
- 左側靠邊裝訂雙線（一粗一細垂直線，幾乎頂滿全頁高）。
- 右上角小圓＋右下偏中大圓，兩者不對稱錯位的裝飾光斑。
- 文字垂直排列：Overline（英文全大寫分類詞）→ H1 主標題（2-3 行結論句）→ 副標說明句 → 落款（提案人/單位/簡報用途）。

### 3.D 流程圖/架構圖策略：Mermaid 先繪圖，再以圖片插入 Gamma

放棄「markdown 內重建精確版面」（研究證實 Gamma 是 AI 生成引擎會二次詮釋，寫死版面容易被誤判成資料表格，弄巧成拙）。改採：

1. Template 產出時，針對 Slide 5/6（1-to-N）或 Slide 4（0-to-1）的流程圖/架構圖，額外生成一段 **Mermaid 語法圖表**（flowchart，節點與連接關係依原有 4 種語意分類：橫向編號步驟卡→用 `-->` 線性鏈、Z 字型技術管線→用 subgraph 分層、指標關係輪盤→視情況簡化為 flowchart TD 放射狀、漏斗圖→用逐層縮小的節點示意）。
2. Mermaid 圖表的視覺風格（顏色、字體、圓角）**必須遵守 `design-system-portfolio-site_6640.md`** 的 token（navy `#153166`、brand-strong `#007583`、暖白背景、圓角 `--rc: 20px` 等）。
3. Gamma Prompt 輸出說明中，指示使用者：「請先用 Mermaid 渲染工具（如 mermaid.live）把這段圖表轉成圖片，再手動插入/替換 Gamma 生成的對應頁面」，作為流程圖頁的正確作法，而不是依賴 Gamma 自己生成圖表。
4. 大綱文字本身仍保留一句自然語言描述該頁圖表在講什麼（給 Gamma 排版文字用），但**不要求 Gamma 自己畫出精確圖表**。

### 3.E 風格指令精簡化 + 修正色階 + Outline-first 提醒

- 修正強調色：`#48B7BD` → `#007583`（Brand Strong，pptx 實測值），補上次文字灰 `#525864`。
- 新增頁尾格式規則：左下角 `[簡報系列名稱] · PAGE [兩位數頁碼]`，新增 Overline 元素規則（H1 正上方一行全大寫分類詞）。
- 精細 hex/字體規則移到「Brand Kit 一次性設定指引」附註（說明使用者可在 Gamma Theme Editor 設一次，而非依賴文字 prompt 解析），生成 prompt 本文只留高層基調宣告（如「延用 Navy+Teal 品牌配色」）。
- 新增「生成後檢查清單」段落：提醒使用者 Gamma 對精細指令的遵循度不保證，生成後需人工巡檢 emoji/色彩/密度。
- 新增「Outline-first」使用說明：提醒使用者貼上 Prompt 後，先審閱 Gamma 產出的中間大綱結構，確認無誤才按下「生成完整簡報」（Gamma 官方認可的修正路徑，比事後改內容更省力）。

### 3.F 檔案架構：共用區塊同步機制

保留兩份完全獨立、可各自複製貼上使用的 template 檔案（使用者決策：不合併、不做程式化引用）。新增 `templates/_shared-modules.md` 作為共用區塊的**維護端單一真實來源**，涵蓋：

- 指標定義與 Few-Shot（NSM/FSM/Volume/Guardrail）
- 信心分級澄清機制（3.A）
- 雙重稽核官規則（3.B）
- Gamma 風格精簡指令 + 色票 + 頁尾/Overline 規則（3.E）
- 標題頁公式（3.C）

兩份 template 內對應區塊**字面複製**這些內容（維持單檔可獨立使用），並在區塊上方加註解標記，例如：

```markdown
<!-- SYNC: _shared-modules.md#信心分級澄清機制 -->
```

未來修改共用邏輯時，先改 `_shared-modules.md`，再手動同步貼回兩份 template 對應區塊。

## 4. 明確不採用的方向（附理由，避免未來重新討論）

- **信心分級的「快速通道」**（spec 已完整時跳過自評）：不採用，維持每次都完整評估的一致性（使用者決策）。
- **在 markdown 裡用表格/巢狀清單重建精確版面**：不採用。Gamma 是 AI 生成引擎會做語意二次詮釋，寫死版面容易被誤判為資料表格；官方路徑是語意標籤 + Outline-first 審閱（追加調研報告 #7 證實）。
- **案例研究深度附錄模式**（pptx 額外的 9 頁/7 頁深度展示）：本次不納入，因超出目前 7 頁/6 頁核心大綱範疇，且非使用者本次明確要求，列為未來可選擴充。
- **合併兩份 template 成單一自動判斷檔**：使用者決策維持兩份獨立檔案。

## 5. 未涵蓋／延伸事項

- 本設計不涉及對 `pm_proposal_presentation.pptx` 或 `design-system-portfolio-site_6640.md` 本身的任何修改，兩者僅作為視覺規範參照來源。
- Mermaid 圖表產出後的實際渲染驗收（視覺比對）留給使用者手動執行，不在本次 template 文字修改範圍內。
